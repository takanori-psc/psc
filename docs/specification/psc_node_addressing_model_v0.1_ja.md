## 1. 目的

PSC Node Addressing Model は、
PSC Fabric 内におけるノード、ポート、および
ルーティングドメインの識別方法を定義する。

このモデルの目的は、
初期の小規模PSCシステムから将来の大規模分散
ファブリックアーキテクチャまで対応可能な、
拡張性・階層性・実装容易性を備えた
アドレス構造を提供することである。

このアドレスモデルにより、
PSC Fabric 環境における効率的なルーティング、
ノード識別、および転送管理が可能になる。

---

## 2. 設計目標

本アドレスモデルは以下の目標を持って設計されている。

・スケーラビリティ  
小規模なローカルPSCシステムから
将来の大規模分散ファブリックまで対応可能とする。

・階層構造  
クラスタおよびファブリックによる
論理的なノードグループ化を可能にする。

・ルーティング効率  
Routing Control Unit (RCU) が効率的に
ルーティング判断を行える構造とする。

・実装容易性  
ハードウェアおよびファームウェアで
効率的に実装できる構造とする。

・将来拡張性  
階層ファブリックやグローバルPSCネットワークなど
将来の拡張に対応できる構造とする。

---

## 3. アドレスの適用範囲

PSCアドレスは以下の識別に使用される。

・PSC Fabric インスタンス
・Fabric 内のクラスタ
・Fabric に接続されたノード
・ノード内のポート

これらの識別子により、
PSC内部モジュールはルーティングおよび
転送処理を実行することができる。

本アドレスモデルを主に利用するモジュールは
以下である。

Resolver  
RCU (Routing Control Unit)  
TMU (Transfer Management Unit)  
Scheduler

---

## 4. Fabric Address Structure

PSC は PSC Fabric 内の通信エンドポイントを識別するために
階層型アドレス構造を採用している。

アドレスは PSC Fabric の物理および論理トポロジーを
表す4つの階層フィールドで構成される。

基本アドレス形式

`Fabric ID : Cluster ID : Node ID : Port ID`

各フィールドは
PSC Fabric トポロジーの異なる階層を表す。

例

`1 : 2 : 15 : 3`

Fabric 1  
Cluster 2  
Node 15  
Port 3  

この階層構造により、
マルチクラスタPSCファブリックにおいて
スケーラブルなアドレス管理と
効率的なルーティングが可能になる。

---

## 5. アドレスフィールド定義

### 5.1 Fabric ID

Fabric ID は
PSC Fabric ドメインを識別する。

大規模構成では
複数のFabricが存在する可能性があるため、
Fabric ID により
独立したPSC Fabricを区別する。

例

Fabric 0  
Fabric 1  
Fabric 2

### 5.2 Cluster ID

Cluster ID は
Fabric 内のノードグループを識別する。

Cluster は以下のような構成単位を表す。

- 物理ラック
- ローカルネットワークグループ
- データセンターセグメント
- エッジクラスタ

クラスタ単位のグループ化により、
ルーティング効率とスケーラビリティが向上する。

### 5.3 Node ID

Node ID は
Cluster 内のPSCノードを識別する。

各ノードは
PSC Fabric に接続されたデバイスを表す。

代表的なノードタイプは以下である。

- CPUノード
- GPUノード
- メモリノード
- ストレージノード
- ネットワークノード
- アクセラレータノード

Node ID は
クラスタ内で一意でなければならない。

### 5.4 Port ID

Port ID は
ノード内の通信エンドポイントを識別する。

ポートはPSC通信インターフェースを表す。

例

- PSC光ポート
- 内部デバイスインターフェース
- 論理通信エンドポイント

Port ID により
PSCノード内部の特定インターフェースへ
ルーティングすることが可能になる。

---

## 6. ルーティングスコープ

ルーティング判断は主に
Routing Control Unit (RCU) によって行われる。

ルーティング範囲は
階層型アドレス構造によって決定される。

ルーティングレベル

- Fabricレベル
- Clusterレベル
- Nodeレベル
- Portレベル

階層型ルーティングにより
大規模PSCファブリックにおいても
効率的なスケーリングが可能になる。

---

## 7. アドレス解決

アドレス解決は
論理ノード識別子を
実際のルーティングパスへ変換する処理である。

この処理は以下のモジュールで分担される。

- Resolver
- RCU
- TMU

典型的な解決手順は以下である。

1. ノード要求が生成される  
2. Resolver が要求を検証する  
3. アドレスがルーティングロジックにより解釈される  
4. RCU が次ホップを決定する  
5. TMU が転送管理を準備する  
6. Scheduler が転送をスケジューリングする  
7. TEU が転送を実行する  

このモデルにより、
中央集権型ボトルネックを作らずに
分散ルーティングが可能となる。

## 8. アドレス例

例 Fabric

Fabric 1

Cluster 0
    CPU Node 0
    GPU Node 1

Cluster 1
    Memory Node 0
    Storage Node 1

例 アドレス

1 : 1 : 0 : 2

Fabric 1  
Cluster 1  
Node 0  
Port 2  

このアドレスは
Fabric 1 の Cluster 1 に存在する
Memory Node の Port 2 を示す。

## 10. まとめ

PSC Node Addressing Model は、
PSC Fabric 環境において
ノード、クラスタ、およびポートを識別するための
階層型アドレス構造を提供する。

このモデルにより
スケーラブルなルーティング、
効率的な転送管理、
および将来の大規模分散PSCシステムへの
拡張が可能となる。
