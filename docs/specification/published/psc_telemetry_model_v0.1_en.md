## 1. Purpose

This document defines the **Telemetry Model** used in PSC Fabric.

PSC adopts a **fabric-centric distributed computer architecture**, where the operational state of the fabric must be observable in order to maintain stability, performance, and operational visibility.

The Telemetry Model defines:

- methods of observing fabric state
- types of telemetry information
- purposes of telemetry usage
- the relationship between Telemetry and the Control Plane

Telemetry functions as the **observation system** of PSC Fabric.

---

## 2. Scope

This document covers:

- the conceptual structure of telemetry within PSC Fabric
- classification of telemetry information
- the relationship between telemetry and the control plane

This document does **not** define:

- telemetry data formats
- telemetry transport protocols
- telemetry data storage systems

These topics will be defined in separate specifications.

---

## 3. Design Principles

PSC Telemetry follows the design principles below.

### 3.1 Observability

PSC Fabric must be observable through telemetry mechanisms so that system state and behavior can be monitored.

### 3.2 Low Overhead

Telemetry must operate with minimal overhead so that fabric performance is not significantly affected.

### 3.3 Distributed Collection

Telemetry data is collected in a **distributed manner** across nodes in the fabric.

### 3.4 Scalability

The telemetry system must scale from small PSC deployments to very large fabric environments.

---

## 4. Telemetry Overview

Telemetry in PSC Fabric is the mechanism used to collect operational state information.

Telemetry is primarily used for:

- fabric state monitoring
- control plane observation input
- operational monitoring
- fault detection

Telemetry provides the observation infrastructure of PSC Fabric.

---

## 5. Telemetry Sources

Telemetry information is generated by nodes within PSC Fabric.

Typical telemetry sources include:

- Compute Nodes
- Memory Nodes
- Storage Nodes
- Fabric Nodes
- Gateway Nodes

Each node generates telemetry information describing:

- its internal state
- relevant fabric conditions

---

## 6. Telemetry Information Types

Telemetry may include various types of operational information.

Examples include:

- Link Status
- Node Status
- Traffic Statistics
- Congestion Indicators
- Error Events

These information types allow the PSC Fabric state to be observed and analyzed.

---

## 7. Telemetry Flow

Telemetry information is collected and used within PSC Fabric.

The basic flow is:

Node Telemetry  
↓  
Telemetry Collection  
↓  
Control Plane Observation  
↓  
Fabric Control Decision  

Telemetry serves as the observation input for the Control Plane.

---

## 8. Telemetry and Control Plane

Telemetry functions as the **observation mechanism** of the Control Plane.

The Control Plane uses telemetry information to perform tasks such as:

- fabric state analysis
- routing adjustments
- congestion control
- fault handling

Telemetry enables the control system to understand the current condition of the fabric.

---

## 9. Telemetry and Management Plane

Telemetry is also used by the **Management Plane** for operational monitoring.

Typical uses include:

- fabric monitoring
- performance analysis
- fault diagnosis
- operational logging

The Management Plane uses telemetry data to understand and manage the operational state of PSC Fabric.

---

## 10. Summary

The PSC Telemetry Model defines the **observation mechanism** of PSC Fabric.

Telemetry provides:

- fabric state observation
- input for control plane decisions
- operational monitoring for the management plane

Through telemetry, PSC Fabric becomes an **observable and adaptive distributed system**.
