## 1. Purpose（目的）

本ドキュメントは PSC Fabric のトポロジーモデルを定義する。

トポロジーモデルは、PSC ノード、スイッチ、およびリンクが
どのように複数のスケールにわたって構成されるかを説明するものである。

対象となるスケールは以下を含む。

- ラックレベルのローカルシステム
- クラスタレベル
- フロアレベル
- グローバル規模の分散環境

本ドキュメントの目的は以下を定義することである。

- トポロジーレベル
- 各レベルの構造的役割
- 接続原則
- スケーラビリティの方向性
- 障害隔離境界

---

## 2. Scope（適用範囲）

本ドキュメントでは以下のトポロジーレベルを扱う。

- Rack topology
- Cluster topology
- Floor topology
- Global topology

本ドキュメントは 構造的なトポロジー設計を対象とする。

以下の内容は本仕様の対象外である。

- 詳細なルーティングアルゴリズム
- パケットフォーマット
- セキュリティプロトコルの詳細
- 物理コネクタの実装

これらは別ドキュメントで定義される。

## 3. Design Principles（設計原則）

PSC Fabric のトポロジーは以下の設計原則に基づいて設計される。

- 階層型スケーラビリティ
- ローカル自律性
- 限定された障害ドメイン
- 段階的拡張
- ポリシー対応型相互接続
- 異種ノード構成のサポート

---

## 4. Topology Levels（トポロジーレベル）

### 4.1 Rack Topology（ラックトポロジー）

Rack topology は
PSC Fabric における
最小の実用的なデプロイ単位である。

1つのラックには通常以下が含まれる。

- compute nodes
- memory nodes
- storage nodes
- accelerator nodes（GPU、AIアクセラレータ、FPGA など）
- ローカル PSC switching elements

Rack topology では以下が重視される。

- 短距離の高帯域リンク
- 低遅延通信
- ローカルトラフィック集約
- 高速な障害隔離

### 4.2 Cluster Topology（クラスタトポロジー）

Cluster topology は
複数のラックを接続し、より大きな協調ドメインを形成する。

Cluster topology では以下を重視する。

- ラック間通信
- 共有リソースアクセス
- スケーラブルなルーティング協調
- クラスタ内ポリシー制御

### 4.3 Floor Topology（フロアトポロジー）

Floor topology は
複数のクラスタを接続し、
より大きな物理サイト内ネットワークを形成する。

Floor topology では以下が重視される。

- より大きな障害ドメイン分離
- リージョントラフィック集約
- 経路多様性の確保
- 管理ドメインの分離

### 4.4 Global Topology（グローバルトポロジー）

Global topology は
地理的に分散した PSC ドメインを接続する。

Global topology では以下が重視される。

- 長距離フェデレーション
- サイト間ポリシー制御
- トラスト認識型接続
- 帯域および遅延の多様性

## 5. Structural Model（構造モデル）

PSC Fabric のトポロジーは
単一の固定された物理形状として定義されない。

代わりに PSC は
規模やデプロイメント目的に応じて
複数のトポロジーパターンをサポートする。

例：

- star-like aggregation
- spine-leaf 構造
- mesh 補助構造
- 階層型フェデレーション

具体的なトポロジーはデプロイレベルによって異なるが、
すべての構造は以下を維持する必要がある。

- 明確な制御境界
- スケーラブルな拡張経路
- 障害隔離構造
- ルーティング整合性

---

## 6. Fault Domain Model（障害ドメインモデル）

各トポロジーレベルは
独自の障害封じ込め境界を持つべきである。

例：

- rack failure は cluster 制御全体を崩壊させてはならない
- cluster failure は floor 全体の動作を崩壊させてはならない
- floor failure は global federation を崩壊させてはならない

したがってトポロジー設計は以下をサポートする必要がある。

- 障害隔離
- 経路再構成
- 劣化動作
- 部分生存運用

---

## 7. Policy and Security Considerations（ポリシーおよびセキュリティ考慮）

トポロジー境界は
ポリシー境界でもある。

異なるトポロジーレベルでは以下のルールが適用され得る。

- ルーティング許可
- 信頼関係
- リソース可視性
- セキュリティフィルタリング
- ドメイン間転送制御

これにより PSC Fabric は
すべてのノード間に均一な信頼関係を要求することなく
スケールすることができる。

## 9. Summary（まとめ）

PSC Fabric トポロジーは
以下のスケールをまたぐ 多層スケーラブル構造として定義される。

- rack
- cluster
- floor
- global

この階層構造により PSC は

- ローカル効率
- 障害隔離
- ポリシー分離
- 長距離分散拡張

を同時に実現することが可能になる。
